MyFlix REST API Documentation

This section provides documentation for the REST API endpoints available in MyFlix.

  • /directors: POST
    • *?:limit: GET
    • */:name: GET
  • /genres POST
    • *?:limit: GET
    • */:name GET
  • /movies POST
    • *?:limit GET
    • */:title GET
  • /users POST
    • */:username GET, DEL, PATCH
    • Endpoint Method Description Request Body Response Body
    /directors POST Adds a new director {"Name": "String", "Bio": "String", "Birth": "String", "Death": "String"} none
    /directors?:limit GET Returns either all directors or a limited amount of directors, if limit parameter is > 0. none {"directors": [{"id": 1, "name": "Francis Ford Coppola", "birthday": "1939-04-07", "deathday": null, "biography": "Francis is awesome!"}]}
    /directors/:name GET Returns a director by name none {"id": 1, "name": "Francis Ford Coppola", "birthday": "1939-04-07", "deathday": null, "biography": "Francis is awesome!"}
    /genres POST Adds a new genre to the database {"id": 1, "name": "Thriller", "description": "Thriller description"} none
    /genres?:limit GET Either returns all genres or a limited amount of genres, if limit parameter is > 0. none {"genres": [{"id": 1, "name": "Thriller", "description": "Thriller description"}]}
    /genres/:name GET Returns a genre by name none {"id": 1, "name": "Thriller", "description": "Thriller description"}
    /movies POST Adds a new movie to the database. Genre and director must be added to the database before adding a movie. {"id":1, "title":"The Godfather", "description":"description", "genre":1, "director":1, "image":"imagelink"} none
    /movies?:limit GET Returns all movies or a limited amount of movies, if limit parameter is > 0. none {"movies": [{"id": 1, "title": "The Godfather", "description": "description", "genre": "Thriller", "director": "Francis Ford Coppola", "image": "imagelink"}]}
    /movies/:title GET Returns a movie. Selects movie by title. none {"id":1, "title":"The Godfather", "description":"description", "genre":"Thriller", "director":"Francis Ford Coppola", "image":"imagelink"}
    /users POST Adds a new user to the database. {"username": "freshmozart", "email": "freshmozart@gmx.com", "password": "beathoven123", "birthday": "1994-09-26", "favourites": null} none
    /users/:username GET Returns a user with the name that is stored in the parameter username. none {"username": "freshmozart", "email": "freshmozart@gmx.com", "password": "beathoven123", "birthday": "1994-09-26", "favourites": null}
    /users/:username DEL Deletes a user with the name that is stored in the parameter username. none none
    /users/:username PATCH Updates user data. JSON in request body can have one or more of the following keys: "password, username, email, birthday, favourites". Values for password, username or email can't be null. {"password": "newpassword456"} none
